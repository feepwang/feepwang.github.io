C++的指针

Posted on Feb 12, 2021

C++普通指针

下面记录几个常用的C++普通指针的初始化：

  1. int *ptr1;
   
int i = 10;
int *ptr2 = &i;

  2. int *ptr3 = new int;
int *ptr4 = new int(100);

const与指针

const修饰符用来表示一个类型的常量。

  • const的一般用法：
//	常量i
const int const_i;
//	常量引用
const int &const_reference_i;
  • const与指针
//	常量指针（指向常量的指针）
const int * ptr_to_const;
//	指针常量
int *const const_ptr;
//	指向常量的指针常量
const int *const const_ptr_to_const;

智能指针

C++11引入了几类新的指针：shared_ptr、unique_ptr、weak_ptr。

  • 对比普通指针的new、delete命令，智能指针能自动管理内存空间的释放。

  • 智能指针可以看作一个系统内置的类模版。

  • 智能指针里面存有一个给定类型的指针，重载了*、->运算符来模拟指针的操作。

shared_ptr

shared_ptr能够共享管理的指针，直到内置的计数器归零才释放内置指针指向的对象的内存空间。

shared_ptr常用的初始化：

std::shared_ptr<int> sptr1 = std::make_shared<int>();
std::shared_ptr<int> sptr2(new int);

这里有一个值得注意的特性：shared_ptr<int>不能使用int类型的普通指针进行赋值。

std::shared_ptr<int> sptr3 = new int;
//	error:不存在从 "int *" 转换到 "std::__1::shared_ptr<int>" 的适当构造函数
//	没有定义普通指针到shared_ptr的隐式转换

先看下面一段错误示例：

bool foo(std::shared_ptr<int>) {
    // function body
    return true;
}

int *ptr = new int(10);
foo(ptr);		//	错误

shared_ptr也有取指针的成员函数get:

std::shared_ptr<int> sptr(new int(19));
//	get能取到sptr中的指针,该指针指向sptr管理的对象
//	注意：取得的指针不受sptr管理
int *ptr = sptr.get();
//	错误：使用ptr手动释放内存空间，sptr的行为未定义。
delete ptr;		//	编译器检测通过

我们能看到使用get函数是有风险的。

所以原则上只能对指针指向的对象进行操作，不要把指针传递到当前作用域以外。

另外，如果使用get到的指针初始化shared_ptr，新智能指针与旧智能指针是互相独立的，即使它们指向相同的对象；此时其中一个指针完成生命周期释放内存，另一个智能指针的行为是未定义的。所以，不能用get得到的指针初始化新的shared_ptr。

const与shared_ptr

与普通指针相似，智能指针与const也有多种搭配使用的组合：

//	指向常量的shared_ptr
std::shared_ptr<const int> sptr_to_const;
//	常量shared_ptr
const std::shared_ptr<int> const_sptr;
//	指向常量的常量shared_ptr
const std::shared_ptr<const int> const_ptr_to_const;
